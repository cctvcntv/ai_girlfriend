# AI女友：真实恋爱模拟

## 项目简介
《AI女友》是一款基于 Web 的高度智能化恋爱模拟游戏。玩家可以从 120 种不同职业、18-80 岁的女性角色中自定义创建自己的女友，通过自然语言对话、送礼、约会、吵架、惊喜、节日庆祝等互动事件，培养感情，经历从相识到结婚生子的完整过程。游戏后端集成大语言模型（如 DeepSeek），每次对话都基于女友的完整人设（职业、年龄、性格、外貌、记忆）生成个性化回应，并实时影响好感度。所有状态自动保存，关闭浏览器后下次打开可继续游戏。

## 主要特性
- **120 种职业**：涵盖各行各业，每种职业有独立的性格倾向（可自定义）。
- **年龄自定义**：18-80 岁自由选择，外貌、性格修饰语、着装风格随年龄动态变化。
- **高度自定义**：可自定义女友姓名、身高、体重、发型、脸型、基础性格。
- **大语言模型驱动**：支持接入 DeepSeek、OpenAI 等兼容 API，对话自然，情感丰富。
- **情感分析与好感度**：AI 回复中隐式包含情感标签（正面/负面/中性），自动调整好感度。
- **记忆系统**：记录所有历史互动，并在后续对话中引用，形成长期记忆。
- **丰富事件**：聊天、送礼（鲜花/巧克力/首饰/书籍/化妆品）、约会（电影院/餐厅/公园/游乐园/海边）、吵架、惊喜、节日庆祝、求婚、生子。
- **自动保存**：每 30 秒自动保存游戏进度，关闭浏览器后下次访问自动恢复。
- **重新相亲**：随时重置当前女友，重新创建（API 配置保持不变）。
- **响应式界面**：适配手机和电脑，顶部头像可点击展开女友信息面板。
- **API 配置管理**：可在创建时或游戏中随时修改 API 密钥、模型、地址等参数。

## 技术栈
- **后端**：Python 3.6+，Flask 框架，Waitress 生产服务器
- **前端**：HTML5，CSS3（Flexbox 响应式布局），原生 JavaScript（ES6）
- **数据持久化**：JSON 文件存储（位于 `saves/` 目录）
- **AI 集成**：支持 OpenAI 格式的 API（如 DeepSeek、OpenAI、通义千问等）

## 安装与运行

### 环境要求
- Python 3.6 或更高版本
- pip 包管理工具

### 安装步骤
1. **克隆或下载项目代码**到本地文件夹。
2. **进入项目目录**：
   ```bash
   cd girlfriend_web
   ```
3. **安装依赖**：
   ```bash
   pip install flask waitress requests
   ```

### 配置文件
项目根目录下需要创建 `config.json` 文件，用于设置默认 API 参数和游戏选项。示例：
```json
{
    "api": {
        "provider": "deepseek",
        "api_key": "your-default-api-key",
        "model": "deepseek-chat",
        "base_url": "https://api.deepseek.com/v1/chat/completions",
        "temperature": 0.8,
        "max_tokens": 200
    },
    "game": {
        "max_memory": 20,
        "interaction_time_increment": 5,
        "age_interval": 60
    }
}
```
- **api_key**：可留空，游戏内创建女友时会要求填写。
- **其他参数**：可在游戏内的 API 设置中动态修改。

### 启动服务器
```bash
python app.py
```
终端显示 `Starting server on http://0.0.0.0:8080` 后，打开浏览器访问 `http://localhost:8080` 即可开始游戏。

## 使用指南

### 首次启动
- 浏览器打开后会自动弹出“创建你的女友”模态框。
- 选择职业、年龄，填写姓名、身高、体重、发型、脸型、性格。
- 在下方 API 设置区域，勾选“使用AI生成对话”，填写你的 API Key 和其他参数（若使用默认配置可留空）。
- 点击“测试连接”确保 API 可用，然后点击“开始游戏”。

### 主界面
- **顶部栏**：左侧显示女友头像和名字，点击可展开/隐藏信息面板；右侧齿轮图标用于修改 API 设置。
- **信息面板**：显示女友所有属性（姓名、年龄、身高、体重、发型、脸型、职业、性格、着装、关系阶段、好感度、相处时间），以及“使用AI生成对话”开关、“改名”和“重新相亲”按钮。
- **对话历史区域**：显示玩家与女友的所有对话及系统消息。
- **输入框**：输入文字聊天，回车或点击“发送”。
- **事件按钮**：快速触发各种互动（聊天、送礼、约会、吵架、惊喜、节日庆祝、求婚）。

### 好感度与关系阶段
- 好感度范围 0-100，初始 50。
- 关系阶段：相亲初识 → 互有好感 → 恋爱中 → 订婚 → 已婚 → 生子，好感度达标后自动进阶。
- 求婚需好感度 ≥75 且处于恋爱中或以上阶段，手动点击“求婚”按钮。
- 生子事件在已婚且好感度 ≥90 时自动触发，弹出恭喜信息并结束游戏（可刷新重新开始）。

### 年龄增长
- 每次互动增加 5 分钟相处时间，累计每 60 分钟（可配置）年龄增加 1 岁。
- 年龄增长会更新性格修饰语（如青春活力 → 成熟稳重）、着装风格，并随机微调体重和发型。

### 自动保存与恢复
- 游戏每 30 秒自动保存到服务器 `saves/` 目录下的 `{user_id}.json` 文件。
- 关闭浏览器后重新打开，会自动加载上次进度，继续游戏。
- 点击“重新相亲”可删除当前进度并返回创建界面（API 配置保留）。

## 常见问题

### 如何获取 API Key？
- DeepSeek：访问 [platform.deepseek.com](https://platform.deepseek.com) 注册并获取 API Key。
- OpenAI：访问 [platform.openai.com](https://platform.openai.com) 获取。

### 为什么测试连接失败？
- 检查 API Key 是否正确。
- 确认 Base URL 是否与模型提供商匹配（例如 DeepSeek 为 `https://api.deepseek.com/v1/chat/completions`）。
- 确保网络能够访问该 API 地址（部分区域可能需要代理）。

### 如何修改默认 API 配置？
- 在游戏主界面点击顶部齿轮图标，修改后保存即可。
- 若未创建女友，可在创建模态框中直接填写。

### 能否使用其他模型？
- 只要兼容 OpenAI 格式即可，将 Base URL 和模型名称改为对应的即可。

### 数据存储在哪里？
- 所有女友数据保存在项目根目录下的 `saves/` 文件夹中，以用户会话 ID 命名的 JSON 文件。
- 删除这些文件将丢失对应进度。

## 开发者扩展
- **增加职业**：在 `game_logic.py` 的 `PROFESSIONS` 列表中添加职业名称即可。
- **修改性格选项**：修改 `PERSONALITY_OPTIONS` 列表。
- **添加新事件**：在 `game_logic.py` 的 `Girlfriend` 类中增加方法，并在前端 HTML 和 JS 中添加对应按钮和请求。
- **更换前端框架**：可替换为 Vue/React，但需调整 API 交互逻辑。
